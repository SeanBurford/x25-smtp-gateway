# X.25-SMTP GATEWAY(8)

## NAME
x25smtp - Bi-directional SMTP to X.25 Gateway

## SYNOPSIS
**x25smtp** [**-a** *local_addr*] [**-P** *cud*] [**-l** *listen_addr*] [**-g** *smtp_gateway*] [**-S** *software*] [**-A** *auth_file*] [**-t** *timeout*] [**-V** *varz_addr*]

## DESCRIPTION
**x25smtp** is a gateway utility that facilitates the exchange of electronic mail between X.25 networks and the Simple Mail Transfer Protocol (SMTP). It supports both inbound and outbound relaying.

When acting as an SMTP-to-X.25 relay, the program listens on a specified TCP port for incoming SMTP connections. Each message received is encapsulated and transmitted over an X.25 circuit to a destination derived from the recipient's address.

When acting as an X.25-to-SMTP relay, the program listens for incoming X.25 calls. Data received over these circuits is converted into SMTP transactions and delivered to a downstream SMTP mail server.

## OPTIONS
**-a** *local_addr*
:   Specify the local X.25 address to use for listening or placing calls. This option is **mandatory**.

**-P** *cud*
:   Specify the Protocol or Call User Data (CUD) in hexadecimal format. Default is `C0F70000`.

**-l** *listen_addr*
:   Enable the SMTP-to-X.25 relay by specifying the TCP address to listen on (e.g., `127.0.0.1:25`).

**-g** *smtp_gateway*
:   Enable the X.25-to-SMTP relay by specifying the destination SMTP server address for messages arriving via X.25.

**-S** *software*
:   The software name to appear in the `Received:` headers generated by the gateway. Default is `x25smtp`.

**-A** *auth_file*
:   Path to a file containing allowed `MAIL FROM` addresses (one per line). If provided, incoming SMTP commands with sender addresses not in this list will be rejected.

**-t** *timeout*
:   Set the receive timeout in seconds for inbound sockets. If no data is received within this period, the connection is closed. Default is `60` seconds.

**-V** *varz_addr*
:   Enable an internal HTTP server at the specified address (e.g., `127.0.0.1:5555`) to export application metrics in the `expvar` format. Metrics are available at `/debug/vars`.

## SIGNALS
**SIGINT**, **SIGTERM**
:   Graceful shutdown. The program will log final statistics before exiting.

**SIGUSR1**
:   Triggers a dump of the current statistics to the log.

**SIGPIPE**
:   Ignored to prevent crashes during socket write errors.

## FILES
*auth_file*
:   A plain text file containing one email address per line. Leading and trailing whitespace is ignored.
